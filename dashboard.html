How the Index
Is Built

Every minute of a match, LiveScore publishes text commentary. Instead of traditional xG or event data, this project treats those sentences as a sensor stream — extracting pressure signals from language patterns and sentiment intensity.

01
Scrape

Selenium scraped minute-by-minute text for 25 WC 2026 matches from LiveScore, handling JS-rendered content.

02
Classify

Each entry classified into 14+ event types (goal, attempt, corner, card…) via layered regex pattern matching.

03
Weight

Events receive base weights (goal=10, red card=8, attempt=3–4) amplified by VADER NLP sentiment urgency.

04
Roll

A 5-minute rolling window sums weighted events per team, producing a continuous Pressure Index time series.

// Pressure Index at minute t, for team T
PI(T, t) = Σ weight(e) × (1 + |VADER(e)|)   ∀ e ∈ window[t-5 : t]
// weight(e) = base_weight assigned by event type classification

Commentary-derived pressure alone doesn't reliably predict goals — but split by goal type, distinct patterns emerge that align with football theory.

  • Sustained pressure doesn't precede most WC 2026 goals — disruption and transition do.
  • Counter-attack goals show +1.89 pre-goal uplift — the index captures tactical imbalance.
  • Set piece goals trend toward lower pre-goal pressure (p=0.15), consistent with dead-ball restarts.
  • Open play goals (60%) show near-zero uplift — goals emerge from chance, not accumulated pressure.
  • Commentary text as a sensor stream is viable — it needs goal-type segmentation to reveal structure.
